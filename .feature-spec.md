# Feature: Create User

## User Story

As a client I want to create a user so that I can use the chat system.

## Requirements

- User creation happens via **HTTP POST `/users`** endpoint (not WebSocket)
- Client provides a **username only** (no password)
- Server returns a **username** and **access_token** on success
- The access token is used to connect to the WebSocket chat
- WebSocket connections remain token-only (no change to auth flow)
- **Remove hardcoded seed users** (foo_user/bar_user) from Setup; only keep the default chat room

## Acceptance Criteria

1. `POST /users` with `{"username": "alice"}` returns `201` with `{"username": "alice", "access_token": "<generated_token>"}`
2. `POST /users` with an existing username returns `409` with `{"error": "username already taken"}`
3. `POST /users` with missing/empty username returns `400` with `{"error": "username is required"}`
4. After creating a user via HTTP, the returned access_token can be used to connect to the WebSocket chat
5. The UI allows creating a user (username input + create button) and then connecting to the chat

## Plan

### 1. Create `CreateUser` use case (`lib/ex_chat/use_cases/create_user.ex`)
- Accepts a `username`
- Calls `UserSessions.create(username)` - if `{:error, :already_exists}`, return `{:error, "username already taken"}`
- Generates an access token via `:crypto.strong_rand_bytes(16) |> Base.url_encode64()`
- Registers token via `AccessTokenRepository.add(token, username)`
- Returns `{:ok, %{username: username, access_token: token}}`
- **Unit test:** `test/ex_chat/use_cases/create_user_test.exs` (mocked, same pattern as `CreateChatRoomTest`)

### 2. Add `POST /users` route to the Router (`lib/ex_chat/web/router.ex`)
- Add `Plug.Parsers` for JSON body parsing (using Poison as decoder)
- Add `post "/users"` route that calls `CreateUser.on(username)`
- Returns 201/409/400 with JSON responses

### 3. Remove seed users from Setup (`lib/ex_chat/setup.ex`)
- Remove `UserSessions.create` and `AccessTokenRepository.add` calls for foo_user/bar_user
- Keep only `ChatRooms.create("default")`

### 4. Update acceptance tests (`test/ex_chat/web/websocket_acceptance_test.exs`)
- Fix the test at line 74 that uses `bar_token` directly (create the user explicitly instead)
- Add acceptance tests for the HTTP `POST /users` endpoint (201, 409, 400 cases)
- Add end-to-end test: create user via HTTP, then connect to WebSocket with the returned token

### 5. Update the UI (`priv/static/chat.html`)
- Add a "create user" screen: username input + create button
- On create: `POST /users`, receive token, then open WebSocket with it
- Hide the chat UI until user is created/connected
